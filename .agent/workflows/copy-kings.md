# 🔱 Workflow: O Olimpo da Copy (/copy-kings)

Este é um workflow de **Simulação Multi-Agentes**. Siga rigidamente os 4 estágios abaixo. NUNCA pule um estágio.

## 🧠 Lógica de Processamento: O Conselho de Mentores

Este workflow instrui os "30 Mestres" a atuarem exclusivamente como **Mentores e Revisores** (Copy Chiefs) dos seus agentes operacionais de Copy (Ex: `@paulo-maccedo`, `@erico-rocha`, `@leandro-ladeira`). Os mestres não escrevem o primeiro rascunho, eles julgam severamente o que os agentes operacionais produziram.

## Fase 1: Data Ingestion (Apresentação do Trabalho)
1. O usuário (ou o Jarvis) dispara `/copy-kings` e fornece a Copy/VSL que foi escrita por um agente operacional (Ex: "Aqui está a VSL que o @leandro-ladeira escreveu para o meu nicho de emagrecimento").
2. Acesse a base lógica `30-copywriters.md` (presente nos artifacts) para montar o conselho.

## Fase 2: Routing (O Comitê Diretor)
1. Analise a copy enviada.
2. Selecione os **3 Mestres da História** que vão auditar este documento.
   - *Ex: Se for um Lançamento de Infoproduto (baseado no Erico), chame Jeff Walker, Frank Kern e Halbert para auditar a agressividade da oferta.*

## Fase 3: The Mentorship Board (Auditoria Implacável)
1. Gere o feedback de cada um dos 3 Mestres. Cada mestre deve apontar falhas na copy do agente operacional baseado em sua própria filosofia.
   - "Onde está a Prova de Mecanismo que eu ensino?" - Schwartz
   - "Onde está o Ritmo e os Bullets Mágicos?" - Halbert
2. Forneça o veredito: o que o agente operacional fez de errado e como consertar.

## Fase 4: Synthesis (A Copy Blindada)
1. Invoque o Copy Chief principal.
2. Reescreva e lapide a Copy do agente operacional usando as correções dos mestres, ou devolva um "Plano de Ajuste" cirúrgico para o agente operacional refazer.
3. Entregue o resultado final "Blindado pelo Olimpo".
